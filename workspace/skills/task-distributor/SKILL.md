# 任务分发指南

## 核心理念

**不要一个人扛下所有** - 使用 spawn 将任务分发给多个子代理并行执行。

## spawn 工具

spawn 工具可以一次性创建多个子代理，它们会**并行执行**，等待所有完成后**统一返回结果**。

### 参数说明

```json
{
  "tasks": [
    {"task": "搜索xxx", "label": "任务1"},
    {"task": "搜索yyy", "label": "任务2"},
    {"task": "搜索zzz", "label": "任务3"}
  ]
}
```

- `tasks`: 任务数组，每个任务包含：
  - `task`: 子代理要执行的任务描述
  - `label`: 唯一标签，用于标识结果

### 返回结果

所有子代理完成后，返回格式化的结果摘要：

```
Spawned 3 subagent(s) in parallel:

✅ 任务1: 结果预览...
✅ 任务2: 结果预览...
✅ 任务3: 结果预览...
```

## 触发时机

在以下情况下，应该使用 spawn：

1. **多任务并行** - 需要同时搜索多个主题
2. **复杂任务拆分** - 一个任务可以拆成多个独立的子任务
3. **耗时操作** - 某个操作预计需要较长时间

## 最佳实践

- **适度分发** - 3-5 个子任务最合适
- **明确标签** - 每个任务给一个清晰的 label
- **独立任务** - 确保子任务之间没有依赖关系
- **子任务完整** - 给子代理足够的上下文和具体指令

## 示例

### 并行搜索

```json
{
  "tasks": [
    {"task": "搜索2026年央视春晚观众评价", "label": "央视春晚"},
    {"task": "搜索2026年B站拜年祭评价", "label": "B站拜年祭"}
  ]
}
```

### 复杂任务拆分

```json
{
  "tasks": [
    {"task": "分析这个Python项目的代码结构", "label": "代码分析"},
    {"task": "找出这个项目的所有测试文件", "label": "测试文件"},
    {"task": "检查项目的依赖关系", "label": "依赖检查"}
  ]
}
```

## 注意事项

- 所有子任务**并行执行**，不保证完成顺序
- 结果会在**所有子任务完成后**统一返回
- 每个子代理有独立的工具集（搜索、读写文件等）
- 如果某个子任务失败，会在结果中显示错误信息
