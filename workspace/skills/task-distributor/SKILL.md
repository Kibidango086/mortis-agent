# 任务分发指南

## 核心理念

**不要一个人扛下所有** - 当任务可以并行处理时，主动使用 subagent 来提高效率。

## 触发时机

在以下情况下，应该考虑使用 subagent：

1. **多任务并行** - 需要同时做多件事（如搜索多个主题、读取多个文件）
2. **复杂任务拆分** - 一个复杂任务可以拆成多个独立的子任务
3. **耗时操作** - 某个操作预计需要较长时间，可以后台执行

## 使用方式

### spawn 工具（推荐）

```go
// 并行启动多个 subagent
spawn tool: "subagent" 
args: {
    "label": "搜索任务",
    "task": "帮我搜索 xxx"
}
```

### subagent 工具

同步调用，会等待结果返回。

## 最佳实践

- **小任务直接做** - 简单任务（如读一个文件）不需要 spawn
- **适度分发** - 3-5 个子任务最合适，不要过度拆分
- **明确指令** - 给 subagent 的任务要清晰具体
- **记得沟通** - subagent 需要用 `message` 工具把结果告诉用户

## 注意事项

- 当前 agent（你）有时候会忘记使用 subagent，这是正常的人类行为 😊
- 当你意识到任务可以并行时，主动提醒用户或直接使用 spawn
- 不要为小任务创建 subagent，オーバーヘッド太大
