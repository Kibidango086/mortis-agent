# 任务分发指南

## 核心理念

**不要一个人扛下所有** - 当任务可以并行处理时，主动使用 spawn 来分发任务。

## 工具说明

### spawn 工具（推荐）

- **同步等待**：spawn 会等待子代理完成任务后再返回
- **返回结果**：子代理完成后，结果直接返回给 LLM
- **适用场景**：大多数任务分发场景

```json
{
  "tool": "spawn",
  "args": {
    "task": "帮我搜索 xxx",
    "label": "搜索任务"
  }
}
```

### subagent 工具

- **同步执行**：与 spawn 类似，等待子代理完成
- **返回值不同**：返回给用户的是简短摘要，返回给 LLM 的是详细结果
- **适用场景**：需要明确区分用户可见内容和 LLM 上下文的场景

### 真正的并行

如果需要真正的并行执行（同时启动多个子任务），需要在 **单次 LLM 调用中多次调用 spawn**。LLM 可以在一次响应中发起多个 tool call，这些会并行执行。

## 触发时机

在以下情况下，应该考虑使用 spawn：

1. **多任务并行** - 需要同时做多件事（如搜索多个主题、读取多个文件）
2. **复杂任务拆分** - 一个复杂任务可以拆成多个独立的子任务
3. **耗时操作** - 某个操作预计需要较长时间

## 最佳实践

- **小任务直接做** - 简单任务（如读一个文件）不需要 spawn
- **适度分发** - 3-5 个子任务最合适，不要过度拆分
- **明确指令** - 给子代理的任务要清晰具体
- **记得沟通** - 子代理需要用 `message` 工具把结果告诉用户

## 注意事项

- spawn 是同步的，会等待子代理完成
- 如果需要真正并行，在一次 LLM 响应中发起多个 spawn 调用
- 不要为小任务创建子代理，开销太大
